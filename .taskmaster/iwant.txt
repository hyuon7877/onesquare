# 팀 관리자용 일정/업무/매출 통합 관리 시스템 개발 요청

## 프로젝트 개요
팀원 및 파트너를 통합 관리할 수 있는 PWA 기반 관리 시스템을 개발하고 싶습니다. 현장에서 파트너가 사용하는 현장 리포트 시스템과 관리자가 사용하는 통합 관리 시스템을 분리하여 각각의 사용성을 극대화하고, 모든 데이터가 Notion API를 통해 실시간으로 동기화되는 완전한 비즈니스 솔루션을 원합니다.

## 시스템 아키텍처

### 2-Tier 시스템 구조
1. 현장 리포트 시스템 (파트너 전용)
   - 모바일 PWA 앱
   - 현장 작업 최적화 UI
   - 오프라인 지원
   - 실시간 데이터 전송

2. 통합 관리 시스템 (관리자 전용)
   - 웹 기반 관리 대시보드
   - 모든 현장 데이터 통합 조회
   - 고급 분석 및 관리 기능
   - Notion DB 완전 연동

### 데이터 흐름
현장 리포트 시스템 (파트너) → 통합 관리 시스템 (관리자) → Notion Database (데이터 저장소)

## 운영 환경
- 운영 시간: 설정 변경 가능 (기본: 일요일 저녁 ~ 목요일 저녁, 주 5일)
- 관리자가 운영 요일/시간 커스터마이징 가능
- 팀별/파트너별 개별 운영시간 설정 지원
- 기준 근무시간: 하루 8시간 (설정 변경 가능)
- 접속 환경: PWA 앱 (모바일 최적화, 오프라인 지원), 웹 브라우저 (PC/모바일 반응형), 앱 설치 지원 (iOS/Android)
- 데이터 저장: Notion Database (실시간 협업 가능)
- 통합 관리: 팀원 + 파트너 + 도급사 통합 관리 시스템

## 사용자 권한 체계 및 인증 시스템

### 인증 방식
- 세션 관리: 세션 쿠키 + CSRF 토큰 기반으로 보안 강화
- 파트너 인증: 휴대폰 OTP (SMS 또는 앱 기반 일회용 비밀번호), 간편 인증 (현장 작업 환경에 최적화), 휴대폰 분실 대응 (관리자가 기존 계정의 휴대폰 번호 변경 기능)
- 관리자/매니저 인증: 이메일 + 비밀번호 (간단하고 효율적인 로그인 방식), 단일 인증 (복잡한 추가 인증 없이 빠른 관리 업무 접근)

### 보안 정책
- 자동 만료 시간 설정 (파트너: 8시간, 관리자: 4시간)
- 동시 로그인 제한 및 이상 접근 감지
- 비밀번호 정책: 관리자는 복합 문자 + 정기 변경 권장, 파트너는 OTP 기반으로 별도 비밀번호 불필요
- 계정 복구: 파트너는 관리자 승인을 통한 휴대폰 번호 변경, 관리자는 이메일 인증을 통한 비밀번호 재설정

### 사용자 권한 그룹

1. 최고 관리자 (Super Admin)
   - 모든 기능 접근 가능
   - 사용자 권한 설정 및 새로운 권한 그룹 생성
   - 시스템 설정 변경 (운영시간, 사용자 유형 등)
   - 전체 데이터 통계 및 분석
   - Notion Database 연동 설정 및 관리
   - 매출 정보 완전 접근: 모든 매출 데이터 조회 및 권한 설정

2. 중간 관리자 (Manager)
   - 담당 팀원/파트너 일정 등록/수정/삭제
   - 담당 그룹 일정 전체 조회
   - 연차 승인/반려
   - 담당 그룹 매출 현황 조회
   - 담당 그룹 업무시간 관리
   - 매출 정보 제한적 접근: 담당 구역/팀원에 한정된 매출 데이터 조회

3. 팀원 (Staff)
   - 개인 일정 조회 및 제한적 수정
   - 연차 신청 기능
   - 개인 대시보드 이용
   - 개인 업무 진행 상태 업데이트
   - 파트너 정보 검색 및 조회: Notion Database 연동 파트너 정보 검색 기능
   - 개인 매출 정보: 관리자가 설정한 권한에 따른 개인/담당 구역 매출 조회

4. 파트너 (Partner)
   - 파트너 전용 대시보드 이용
   - 파트너 일정 조회 및 제한적 수정
   - 파트너 매출 현황 조회 (권한 범위 내)
   - 도급 관련 업무 관리
   - 개인 매출 정보: 본인 관련 매출 데이터만 조회

5. 도급사 (Contractor)
   - 도급 프로젝트 관련 일정 조회
   - 도급 업무 진행 상태 관리
   - 도급 매출 및 정산 관련 데이터 접근
   - 프로젝트별 매출: 담당 프로젝트 관련 매출 정보만 조회

6. 커스텀 권한 그룹
   - 관리자가 필요에 따라 새로운 권한 그룹 생성 가능
   - 세부 권한 조합으로 맞춤형 접근 권한 설정
   - 매출 정보 권한 세분화: 그룹별로 매출 정보 접근 범위 개별 설정

## 주요 기능 요구사항

### 1. 사용자 관리 시스템 (확장형)
- 다중 사용자 유형 관리: 팀원, 마스터, 도급사 + 커스텀 유형
- 권한 그룹 관리: 새로운 권한 그룹 생성 및 세부 권한 설정
- 팀/부서/파트너 구분: 다계층 조직 구조 지원
- 역할별 접근 제어: 세밀한 기능별/데이터별 접근 권한 관리
- 사용자 유형 변경: 실시간 권한 변경 및 이력 관리

#### 파트너 정보 관리 시스템 (Notion Database 연동)
- Notion 파트너 DB 연동: 실시간 동기화, 자동 업데이트, 단방향/양방향 동기화 선택
- 팀원용 파트너 검색 기능: 통합 검색 (파트너명, 연락처, 담당 구역, 전문 분야 등), 상세 정보 조회, 빠른 연락 (전화, 문자, 이메일 바로 연결), 태그 필터링, 실적 현황
- 파트너 정보 표시: 프로필 카드, 담당 구역, 평가 현황, 연락 이력

### 2. 통합 캘린더 및 일정 관리 시스템

#### 일정 CRUD (권한별 차등 적용)
- 등록: 날짜, 시간, 업무 내용, 담당자 지정, Notion 관리공간 연계, 매출 정보 자동 연동 (권한 설정에 따라 표시)
- 조회: 권한에 따른 조회 범위 제한
- 수정: 실시간 수정 및 이력 관리
- 삭제: 삭제 권한 및 삭제 로그 관리

#### 고급 일정 기능
- 빠른 일정 등록: 관리자용 간편 등록 인터페이스
- 일정 템플릿: 반복 업무에 대한 템플릿 제공
- 일정 복사: 유사 일정 빠른 생성
- 일정 충돌 감지: 중복 일정 자동 감지 및 알림

#### Notion Database 연동 일정 관리
- 관리공간 연계 시스템: Notion 관리공간 DB 연동, 관리공간 검색 (자동완성), 관리공간 정보 표시, Notion 링크
- 매출 정보 자동 연동 (권한 기반 표시): 매출 데이터 자동 가져오기, 예상 매출 표시, 매출 히스토리, 권한별 매출 정보 제어 (완전 공개/부분 공개/비공개/요약만 공개)

#### 통합 캘린더 시스템 (다중 뷰 지원)
- 캘린더 뷰 옵션: 주간/일간 캘린더 (FullCalendar 라이브러리 기반), 월별 캘린더, 파트너 전용 캘린더
- 업무 스케줄링 시스템: 정기 업무 설정 (자유 주기 설정, 특정 요일/날짜 설정, 반복 횟수 및 유효 기간 설정), 1회성 업무 설정 (특정 날짜 임시 업무, 긴급 점검/특별 행사, 개별 파트너 할당)
- 캘린더 상호작용 기능: 일정 드래그 & 드롭, 실시간 업데이트, 팀원 배정 및 색상 구분, 연차 표시
- 필터링 및 검색 시스템: 전체 일정 보기/팀원별 필터링, 관리공간별 필터링, 프로젝트별/업무 유형별 필터링, 연차 필터링
- 업무 진행 상태 관리: 칸반 보드 (예정/진행중/완료/보류), 드래그앤드롭 상태 변경, 우선순위별 정렬, 완료 상태 표시
- 통계 대시보드: 팀원별 일정 현황, 팀원별 매출 현황 (권한 기반), 현장점검 관리
- 필수 업무 체크 시스템: 진행 차단, 체크 확인, 상태 추적

### 3. 연차 관리 시스템
- 연차 신청: 팀원이 연차 신청 (사유, 기간)
- 승인 프로세스: 관리자 승인/반려 시스템
- 연차 현황: 개인별 연차 사용 현황 및 잔여 연차
- 연차 달력: 팀 전체 연차 현황 달력 뷰
- 알림 시스템: 신청, 승인, 반려 시 실시간 알림

### 4. 업무시간 관리 시스템
#### 시간 추적 및 관리
- 기준 시간 설정: 하루 8시간 (관리자 설정 변경 가능)
- 실제 근무시간 기록: 시작시간, 종료시간 입력
- 시간 초과/미달 표시: 8시간 미만 (주황색), 8시간 정확 (녹색), 8시간 초과 (빨간색)

#### 업무시간 통계
- 일별 통계: 팀원별 일일 근무시간
- 주별 통계: 주간 총 근무시간 및 평균
- 월별 통계: 월간 총 근무시간 및 목표 달성률
- 엑셀 내보내기: 데이터 추출 및 분석용

### 5. 매출 관리 시스템 (Notion Database 연동 - 권한 기반)
#### Notion 매출 데이터 연동 (읽기 전용)
- 실시간 매출 데이터 동기화: Notion Database 연동, 자동 업데이트, 단방향 동기화

#### 매출 정보 권한 관리 시스템
- 권한별 매출 정보 접근 제어: 완전 접근, 부서별 접근, 개인별 접근, 요약 접근, 접근 금지
- 세밀한 권한 설정: 일별 매출, 월별 매출, 매출 히스토리, 예상 매출, 매출 비교

#### 매출 통계 및 리포트 (권한 기반 표시)
- 권한별 매출 분석: 일별/월별/팀원별 매출 분석, 매출 차트
- 매출 정보 마스킹: 금액 마스킹, 비율 표시, 범위 표시

### 6. 알림 및 대시보드 시스템
#### 실시간 알림 기능
- 일정 관련: 일정 등록, 수정, 삭제, 충돌 알림
- 연차 관련: 연차 신청, 승인, 반려 알림
- 업무시간 관련: 근무시간 부족/초과 알림
- 시스템 알림: 중요 공지사항
- PWA 푸시 알림: 모바일 푸시 알림 및 배지 표시

#### 개인 대시보드 (팀원용)
- 개인 일정: 금일/금주 개인 일정
- 근무 현황: 개인 근무시간 현황
- 연차 현황: 잔여 연차 및 사용 내역
- 개인 매출: 개인 매출 현황 (권한 범위 내)
- 파트너 빠른 검색: 자주 연락하는 파트너 빠른 접근

### 7. Notion Database 통합 뷰어 (동적 확장)
#### 기존 Notion DB 연동
- 업무현황판: 실시간 업무 진행 상태 대시보드
- 파트너현황: 파트너별 매출, 일정, 성과 통합 뷰
- 관리공간: 운영 관련 설정 및 관리 데이터
- 파트너 정보: 파트너별 상세 정보, 연락처, 담당 구역, 실적 데이터

#### 동적 Database 연동 시스템
- DB 추가 인터페이스: 관리자가 UI에서 새로운 Notion Database 추가
- 자동 스키마 감지: Notion DB 구조를 자동으로 분석하여 필드 매핑
- 커스텀 뷰 생성: 추가된 DB에 대한 커스텀 대시보드 자동 생성
- 실시간 연동: 새로 추가된 DB도 실시간 동기화 적용
- 권한 매핑: 새 DB에 대한 권한별 접근 제어 자동 설정

#### 통합 뷰어 기능 (확장형)
- 임베디드 뷰: 앱 내에서 모든 연동된 Notion 페이지 직접 조회
- 별도 창 모드: 팝업 또는 새 탭으로 Notion 페이지 열기
- 다중 DB 대시보드: 여러 Notion DB를 하나의 대시보드에 통합 표시
- 필터링 연동: 앱 내 필터 설정이 모든 연동된 Notion 뷰에 반영
- 실시간 동기화: 모든 연동된 Notion 변경사항 실시간 반영
- 권한별 접근: 사용자 권한에 따른 각 Notion DB별 접근 제어

### 8. 텔레그램 봇 연동 시스템 (선택적 설정)
#### 쉬운 설정 인터페이스
- 봇 토큰 설정: 관리자 UI에서 텔레그램 봇 토큰 간단 입력
- 기능별 ON/OFF: 일정 알림, 연차 알림, 업무시간 알림
- 사용자별 설정: 개인별로 받을 알림 유형 선택 가능
- 명령어 커스터마이징: 관리자가 봇 명령어 활성화/비활성화

#### 핵심 기능 (선택적)
- 실시간 알림: 선택된 이벤트만 텔레그램으로 푸시
- 명령어 기반 조회: /today (오늘 일정), /schedule (주간 일정), /leave (연차 현황), /my_sales (개인 매출), /partner (파트너 검색)
- 권한별 접근: 사용자 권한에 따른 차등 명령어 제공
- 설정 UI: 웹 인터페이스에서 봇 설정 전체 관리

### 9. 멀티미디어 피드백 시스템
#### 양방향 소통 시스템
- 실시간 피드백 교환: 파트너 ↔ 관리자 간 실시간 피드백 교환
- 스레드형 대화: 하나의 피드백에 대한 연속적인 대화 지원
- 피드백 분류: 업무 관련, 개선 제안, 문제 신고 등 카테고리별 분류
- 우선순위 설정: 피드백의 긴급도 및 중요도 설정

#### 멀티미디어 피드백 기능
- 파일 업로드 지원: 사진 첨부 (JPEG, PNG, WEBP), 동영상 첨부 (MP4, AVI, MOV), 문서 첨부 (PDF, DOC, XLS)
- 미리보기 기능: 업로드된 이미지/동영상 인라인 미리보기
- 버전 관리: 수정된 파일의 버전 히스토리 관리
- 자동 압축: 관리자 설정에 따른 파일 크기 최적화

#### 피드백 수집 및 관리
- 360도 피드백: 상사, 동료, 하급자로부터의 다면 평가
- 실시간 피드백: 업무 완료 시점의 즉시 피드백 수집
- 비주얼 피드백: 이미지/동영상 기반 업무 결과물 피드백
- 개선 제안: 멀티미디어를 활용한 구체적 개선 아이디어 수집
- 문제 신고: 현장에서 발생한 문제 상황을 사진/영상과 함께 신고
- 작업 완료 인증: 완료된 작업을 사진/영상으로 인증

#### 피드백 상태 관리
- 상태 추적: 접수/확인/처리중/완료 단계별 상태 관리
- 담당자 배정: 피드백 내용에 따른 담당자 자동 배정
- 처리 기한: 피드백 유형별 처리 기한 설정 및 알림
- 만족도 체크: 피드백 처리 후 만족도 조사

#### 분석 및 리포트 (멀티미디어 포함)
- 피드백 통계: 유형별, 부서별, 개인별 피드백 현황
- 멀티미디어 분석: 업로드된 이미지의 분류 및 태깅
- 텍스트 분석: AI 기반 피드백 내용 감정/키워드 분석
- 비주얼 리포트: 이미지/동영상을 포함한 종합 피드백 리포트
- 트렌드 분석: 시간별 피드백 패턴 및 이슈 트래킹
- 개선 효과: 피드백 반영 후 개선 효과 측정

### 10. 현장 리포트 시스템 (Tally 폼 완전 대체)

## 현장 리포트 시스템 (파트너 전용 PWA 앱)

### 시스템 구조 및 데이터 흐름
- 파트너 → 현장 리포트 시스템: 모바일 PWA 앱에서 리포트 작성
- 현장 리포트 → 통합 관리 시스템: 실시간 데이터 전송 및 동기화
- 통합 관리 → Notion Database: 자동 데이터 연동 및 백업
- 양방향 동기화: 모든 변경사항이 실시간으로 각 시스템에 반영

### Tally 폼 기능 재현 및 확장
- 기존 Tally 폼 분석: 현재 사용 중인 폼 구조 완전 재현
- 동일한 필드 구조: 기존 입력 항목들을 그대로 유지
- 추가 기능 확장: 사진 첨부 (자동 압축 1MB), 동영상 첨부, 자동 시간 기록

### 현장 업무 흐름 및 UI 순서
1. 본사 공지사항: 고정 공지 + 일반 공지 확인
2. 오늘의 업무 확인: 정기 업무 및 1회성 업무 체크
3. 업무 시작 버튼: 가장 우선적으로 표시
4. 체크리스트: 필수/선택 항목 구분하여 확인
5. 예시 사진 보기: 사진 촬영 전 가이드 확인
6. 사진 촬영: 관리자 설정에 따른 자동/원본 압축
7. 비품 재고 체크: 사진 촬영 후 비품 상태 확인
8. 추가 내용 작성: 기타 특이사항 기록
9. 리포트 전송: 통합 관리 시스템으로 즉시 전송

### 본사 공지사항 시스템
- 고정 공지사항: 상단 고정, 고정 설정, 필수 확인, 강조 표시
- 일반 공지사항: 자유 공지, 시간순 정렬, 읽음 상태, 자동 만료

### 업무 시간 관리 시스템
- 업무 시작 버튼: 파트너가 버튼 클릭 시 시작 시간 자동 기록
- 자동 종료 시간: 리포트 전송 완료 시 종료 시간 자동 설정
- 수동 시간 수정: 리포트 전송이 늦어진 경우 수동으로 시간 수정 가능
- 권한별 설정: 관리자가 파트너별로 시간 기록 기능 사용/미사용 설정
- 시간 검증: 비합리적인 시간 입력 시 경고 메시지

### 체크리스트 관리 시스템
- 필수 항목: 필수 완료 (빨간색 표시), 진행 차단, 관리자 설정
- 선택 항목: 권장 사항 (파란색 표시), 선택 완료, 통계 활용
- 동적 관리: 실시간 변경, 템플릿별 설정

### 비품 관리 기능
- 비품 재고 체크: 재고 현황, Notion 연동, 재고 상태, 이력 관리
- 사용량 기록 (선택적): 관리자 설정, 사용량 입력, 소비 패턴 분석, 선택적 적용

### 데이터 보존 및 최적화 시스템
- 자동 저장: 임시 저장, 자동 복구, 오프라인 지원
- 미디어 압축 최적화: 이미지 압축 (기본 1MB), 동영상 압축, 압축 설정, 원본 옵션, 빠른 전송, 재시도 로직

### 공간별 맞춤 리포트 시스템
- 공간별 리포트 템플릿: 각 업무 공간별로 다른 리포트 양식
- 예시 사진 시스템: 촬영 가이드, 관리자 관리, 단계별 가이드, 품질 기준

### 공간 정보 및 중요사항 표시
- 공간 기본 정보: 공간명 및 주소, 연락처, 운영 시간, 특이사항
- 중요사항 관리: 실시간 알림, 관리자 편집, 우선순위 표시, 유효기간 설정

### 현장 전용 기능
- 모바일 최적화: PWA로 모바일에서 쉬운 입력
- 리포트 전송 확인: 전송 상태, 전송 이력, 재전송 기능, 완성도 체크
- 현장 맞춤 UI: 장갑 착용 상태에서도 쉬운 터치 인터페이스
- 빠른 입력: 반복 업무에 대한 원터치 입력 기능
- 템플릿 시스템: 반복 업무에 대한 빠른 입력 템플릿
- 자동 완성: 이전 입력 기록 기반 자동 완성 기능
- 실시간 피드백 수신: 관리자 피드백을 현장에서 즉시 확인

## 11. 통합 관리 시스템 (관리자 전용 웹 대시보드)

### 관리자 대시보드 및 AI 분석 시스템
#### 관리자 메인 대시보드
- 팀 전체 현황: 실시간 팀 상태 요약
- 오늘의 일정: 팀 전체 금일 일정 현황
- 매출 현황: 실시간 매출 현황 (권한에 따른 매출 정보 표시)
- 현장점검 관리: 통합 캘린더 시스템의 현장점검 관리 기능과 연동
- 알림 센터: 미확인 알림 목록
- 빠른 액세스: 주요 기능 바로가기

#### AI 데이터 분석 기능
- 매출 예측: 과거 데이터 기반 매출 트렌드 분석 및 예측 (권한 범위 내)
- 업무 효율성 분석: 팀원별 업무 패턴 및 생산성 분석
- 일정 최적화: AI 기반 최적 일정 배치 추천
- 이상 패턴 감지: 급격한 매출 변화, 비정상적 근무 패턴 감지

#### AI 인사이트 시스템
- 성과 분석: 개인별/팀별 성과 지표 분석 및 개선 제안
- 리소스 최적화: 인력 배치 및 업무 분배 최적화 추천
- 위험 요소 예측: 번아웃 위험, 이직 가능성 등 예측 알림
- 목표 달성 확률: 현재 진행도 기반 목표 달성 확률 계산

#### AI 시각화 대시보드
- 대화형 차트: 클릭/드릴다운 가능한 인터랙티브 차트
- AI 리포트: 자동 생성되는 주간/월간 인사이트 리포트
- 예측 모델링: 다양한 시나리오 기반 결과 예측 시뮬레이션

### 현장 리포트 통합 뷰
- 실시간 수신: 파트너가 전송한 현장 리포트 즉시 확인
- 대시보드 통합: 메인 대시보드에서 모든 현장 리포트 현황 조회
- 리포트 상세 보기: 사진, 체크리스트, 비품 현황 등 모든 정보 통합 표시
- 진행 상황 모니터링: 파트너별 리포트 제출 현황 실시간 추적

### 고급 품질 관리 (관리자 → 파트너)
- 사진별 피드백: 특정 사진에 직접 피드백 작성
- 피드백 매칭: 파트너가 어떤 사진에 대한 피드백인지 명확히 표시
- 인라인 댓글: 사진 위에 직접 댓글 및 마킹 기능
- 피드백 알림: 피드백 작성 시 해당 파트너 현장 리포트 앱에 즉시 알림
- 피드백 이력: 사진별 피드백 히스토리 관리

### 관리자 제어 기능
- 현장 업무 관리: 정기/1회성 업무 스케줄 설정 및 현장 전송
- 체크리스트 관리: 필수/선택 항목 설정 및 현장 앱 실시간 반영
- 예시 사진 관리: 공간별 예시 사진 라이브러리 관리 및 현장 배포
- 공지사항 관리: 고정/일반 공지사항 작성 및 현장 앱 푸시
- 비품 설정: 비품 목록 및 사용량 기록 기능 설정
- 미디어 설정: 이미지/동영상 압축 용량 설정
- 매출 정보 권한 설정: 사용자별/그룹별 매출 정보 접근 권한 관리

### Notion Database 연동 시스템
#### 자동 데이터 전송
- 실시간 동기화: 현장 리포트 → 통합 관리 → Notion DB 자동 전송
- 데이터 매핑: 현장 리포트 필드와 Notion DB 속성 자동 매핑
- 파일 업로드: 사진/동영상도 Notion 페이지에 자동 첨부
- 상태 동기화: 완료/미완료/진행중 상태 실시간 반영

#### 데이터 무결성 보장
- 중복 방지: 동일 리포트 중복 전송 방지 시스템
- 오류 복구: Notion API 장애 시 로컬 큐잉 후 자동 재전송
- 데이터 검증: 필수 필드 누락 시 알림 및 재전송 요청
- 이력 관리: 모든 전송 이력 및 오류 로그 관리

#### Notion DB 구조 최적화
- 동적 스키마: 현장 리포트 구조 변경 시 Notion DB 자동 업데이트
- 관계형 데이터: 파트너, 공간, 업무 등 관련 데이터 자동 연결
- 태그 시스템: 자동 태깅으로 데이터 분류 및 검색 최적화
- 백업 전략: 중요 데이터의 자동 백업 및 복구 시스템

- Notion 연동 관리: 새로운 Notion Database URL 추가/제거, 연동 DB별 동기화 설정, 권한별 DB 접근 제어 설정, DB별 표시명 및 아이콘 커스터마이징

## 기술 요구사항
- Backend: Django (Python) + Django REST Framework
- Database: Notion API (하이브리드 구조, 읽기 전용, 유연한 스키마 매핑) - Notion Database를 메인 데이터 저장소로 활용하되 DB 구조 변경에 자동 적응, Django 내부 캐싱 시스템으로 성능 최적화, 통합 관리 시스템에서는 읽기 전용
- Authentication: Django 기본 인증 + 커스텀 권한 시스템
- API Integration: Notion API SDK (읽기 전용, 동적 스키마 감지), Telegram Bot API
- Frontend (PWA): Django + Service Worker + 모듈화된 컴포넌트
- CSS Framework: Bootstrap 5 + 커스텀 테마 시스템
- JavaScript: Vanilla JS + jQuery + 모듈화된 스크립트 구조
- Charts: Chart.js (매출 통계용) + D3.js (고급 시각화)
- Calendar: FullCalendar.js + 커스텀 뷰 지원
- PWA Features: 오프라인 동작 지원, 푸시 알림 시스템, 앱 설치 가능, 백그라운드 동기화
- Container: Docker + Docker Compose
- Web Server: Nginx (HTTPS 필수 - PWA 요구사항)
- Application Server: Gunicorn
- Static Files: WhiteNoise + Service Worker 캐싱
- Message Queue: Celery + Redis

## 핵심 구현 요구사항:
- **유연한 스키마 매핑**: Notion DB 필드명/타입 변경에 자동 적응
- **동적 데이터 변환**: 런타임에 Notion 스키마 감지 후 Django 호환 형태로 변환  
- **무정전 연동**: 스키마 변경 시에도 서비스 중단 없이 지속적 동기화
- **관리자 친화적**: 새로운 Notion Database 추가 시 ID만 입력하면 자동 연동
- **성능 최적화**: 스키마 정보 캐싱으로 불필요한 API 호출 최소화
